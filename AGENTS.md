# Project Instructions

This project is a professional landing page and product catalog website for a printing and packaging business.

## Business goal

The website must present the company, showcase product categories, display catalog products with “price from” values, and allow customers to request a custom quote.

This is NOT an e-commerce store.

## Technical stack

- Next.js App Router
- TypeScript
- Tailwind CSS
- shadcn/ui
- Supabase PostgreSQL, planned for later
- Supabase Storage, planned for later
- Server Actions or Route Handlers for quote requests, planned for later

## Design rules

- Use DESIGN.md in the project root as the visual source of truth.
- Follow its color palette, spacing, layout, card, button, input, typography and responsive behavior.
- Do not copy HP branding, HP logos, HP names or proprietary identity.
- Use the design only as inspiration for a professional product catalog / marketing website.
- The final website must feel appropriate for a printing and packaging business.

## Product rules

- Do not create a shopping cart.
- Do not create checkout.
- Do not create payment integration.
- Do not create user login.
- Do not create an admin panel yet.
- Use typed mock data first.
- Prepare the code so mock data can later be replaced by Supabase.

## Required sections

- Header with navigation
- Hero section
- Product categories section
- Featured product catalog
- Benefits/services section
- Quote request form
- Process/how-it-works section
- Contact section with WhatsApp and email
- Footer

## Code quality

- Use reusable components.
- Keep files small and organized.
- Use TypeScript types.
- Use accessible semantic HTML.
- Keep the website responsive.
- Avoid overengineering.
- Do not rewrite unrelated files.
- Explain changes briefly after implementing them.
